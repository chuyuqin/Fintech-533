Strategy logic

The final strategy rule set is now fixed instead of optimized over a grid. Daily bars define the setup: the breakout level is the highest high of the prior 20 trading days, yesterday's close must be above the 50-day EMA, and yesterday's volume must be at least 80% of its 20-day average. The intended execution is to buy the first clean breakout, risk 1% of equity, place the stop at 1.2 ATR, set the target at 2.4 ATR, and exit after 5 trading days if neither threshold is hit.

Asset selection

I'm switching the asset-selection process to individual stocks rather than ETFs. The quantitative layer screens a liquid large-cap universe under the same breakout rule set and compares out-of-sample Sharpe ratio, out-of-sample return, drawdown, and trade count. The qualitative layer then breaks ties by asking whether the stock has deep liquidity, sustained institutional participation, and a believable fundamental narrative that can support trend persistence without making the name too dominated by one-off event risk. In other words, I want a stock that both backtests well and makes market-structure sense for breakout trading. Breakout definition and assumptions

The breakout detection function is implemented in Python in breakout_report_generator.py. The parameters are explicit and easy to locate at the top of the script. The revised code path uses IBKR data through shinybroker, assumes 2 bps slippage per side, $0.005 per share commissions with a $1 minimum, and sizes positions so that the initial stop risks about 1% of current equity. The annual risk-free rate in the Sharpe ratio calculation is 2%.
